## User entity relevant columns (src/BairroNow.Api/Models/Entities/User.cs)
Required / relevant fields for export auth:
- Id (Guid, PK)
- Email (string, lowercase, unique)
- PasswordHash (BCrypt hash)
- EmailConfirmed (bool) — NOT enforced by export endpoint
- IsVerified (bool) — NOT enforced by export endpoint (no [VerifiedOnly])
- IsActive (bool, default true)
- AcceptedPrivacyPolicy (bool)
- AcceptedPrivacyPolicyVersion (int, default 1)
- CreatedAt, UpdatedAt (DateTime, required)
- LastExportAt (DateTime?) — the 24h gate, nullable

Export endpoint only requires: `[Authorize]` (valid JWT). No EmailConfirmed, no IsVerified, no admin.

## Export endpoint contract (Controllers/v1/AccountController.cs)
```
GET /api/v1/account/export
Authorization: Bearer <jwt>
[EnableRateLimiting("authenticated")]  // sliding window 100/min — NOT the 24h gate
```
Gate that matters: `if (user.LastExportAt != null && user.LastExportAt > DateTime.UtcNow.AddHours(-24)) return 429;`
**Note:** AccountService.BuildExportAsync is presumed to set LastExportAt. If it does NOT, the 2nd call won't return 429 and this plan MUST either (a) update LastExportAt via SQL between calls, or (b) acknowledge the gap. Task 3 verifies this empirically.

Response: `application/json` file named `bairronow-dados-pessoais.json` (backend) but frontend overrides download attribute to `bairronow-meus-dados.json`.

## Frontend → backend URL resolution (frontend/src/lib/api.ts:11)
```ts
baseURL: process.env.NEXT_PUBLIC_API_URL || 'http://localhost:5000'
```
launchSettings.json default ports are 5041 (http) / 7250 (https) — NOT 5000. Strategy: override with `dotnet run --urls "http://localhost:5000"` so the hardcoded frontend fallback works without rebuilding frontend or touching .env files.

## Auto-migration on startup (src/BairroNow.Api/Program.cs:232-235)
```csharp
db.Database.Migrate();
if (!db.Bairros.Any()) { db.Bairros.AddRange(VilaVelhaBairros.All); db.SaveChanges(); }
```
So a clean `dotnet run` against an empty BairroNow_Dev DB will create the schema and seed Vila Velha bairros. No `dotnet ef database update` invocation needed — if the local SQL user can CREATE DATABASE, it's automatic. If the DB must pre-exist, use sqlcmd CREATE DATABASE as a pre-step.

## Rate-limit gotcha (critical for Task 3)
The "authenticated" SlidingWindowLimiter in Program.cs is 100 req/min — NOT the 24h LGPD gate. The 24h gate is DB-based (User.LastExportAt). Therefore:
- First click from a fresh seeded user → 200, JSON blob, sets LastExportAt
- Second click within seconds → 429 via DB check
This is what we want to verify. No need to hit the sliding-window 100/min limiter.
