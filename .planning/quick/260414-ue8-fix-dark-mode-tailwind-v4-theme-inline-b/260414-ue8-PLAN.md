<objective>
Fix BairroNow Tailwind v4 dark mode regression (UAT Test 3 FAIL, 2026-04-15). The `.dark` class is applied correctly to `<html>` and CSS vars are overridden, but `@theme inline` in globals.css bakes literal hex values into Tailwind utility class output at build time, so utilities like `.bg-bg`, `.bg-muted`, `.border-border`, `.text-fg` emit literal colors (e.g. `background-color: #ffffff`) and do NOT respond to runtime CSS variable changes.

Tailwind v4 docs confirm: plain `@theme` generates utilities that reference `var(--color-*)` (runtime-overridable); `@theme inline` "will use the theme variable value instead of referencing the actual theme variable" (baked at build, not overridable). Verified via https://tailwindcss.com/docs/theme on 2026-04-14.

Purpose: Dark mode must visually apply across the whole app — a visible v1.0 polish regression. This is the last blocker to mark UAT Test 3 PASS.

This single change makes Tailwind v4 emit `background-color: var(--color-bg)` etc. in generated utility classes instead of literal hex, so the `.dark { --color-bg: #0f172a; ... }` overrides at line 40 cascade to every utility.

Do NOT change anything else in globals.css. Leave the `html, body { background: var(--color-bg); ... }` block alone (redundant safety net, harmless). Leave the `* { box-shadow: none !important; }` rule. Leave the `.dark { ... }` block values unchanged.
