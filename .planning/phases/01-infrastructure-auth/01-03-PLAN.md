    **IAuthService.cs:**
    ```csharp
    public interface IAuthService
    {
        Task<(AuthResponse? Response, string? Error)> RegisterAsync(RegisterRequest request, string ipAddress);
        Task<(AuthResponse? Response, string? RefreshToken, string? Error)> LoginAsync(LoginRequest request, string ipAddress);
        Task<(AuthResponse? Response, string? NewRefreshToken, string? Error)> RefreshAsync(string refreshToken, string ipAddress);
        Task LogoutAsync(string refreshToken, string ipAddress);
        Task LogoutAllAsync(Guid userId);
        Task<string?> ForgotPasswordAsync(string email);
        Task<bool> ResetPasswordAsync(string token, string email, string newPassword);
    }
    ```

    **AuthService.cs:**
    - Inject AppDbContext, ITokenService, IEmailService, ILogger
    - RegisterAsync: check email unique, hash password with BCrypt.HashPassword, create User with AcceptedPrivacyPolicy=true/Version=1, generate EmailConfirmationToken (Guid), call IEmailService.SendConfirmationEmailAsync
    - LoginAsync: find user by email, check lockout (LockoutEnd > UtcNow -> return error "Conta bloqueada temporariamente. Tente novamente em 15 minutos."), verify BCrypt.Verify, on failure increment FailedLoginAttempts, on 5th failure set LockoutEnd = UtcNow.AddMinutes(15). On success: reset FailedLoginAttempts=0, generate tokens, store hashed refresh token in DB
    - RefreshAsync: find token by hash in DB, check not revoked and not expired. If revoked (token reuse), revoke ALL tokens for that user (security measure). Otherwise: revoke old, create new with ReplacedByTokenId link, return new auth response
    - LogoutAsync: find and revoke the specific refresh token
    - LogoutAllAsync: revoke all refresh tokens for userId
    - ForgotPasswordAsync: find user by email, generate PasswordResetToken (Guid.NewGuid), set PasswordResetTokenExpiry = UtcNow.AddHours(1), call IEmailService.SendPasswordResetEmailAsync. Always return success (don't reveal if email exists)
    - ResetPasswordAsync: find user by email, verify token matches and not expired, hash new password, clear reset fields

    Store refresh tokens hashed: `BCrypt.HashPassword(rawToken)` on create, `BCrypt.Verify(rawToken, storedHash)` on lookup. Actually — BCrypt is slow for lookup (must check each row). Better approach: store SHA256 hash for lookup, which is fine for refresh tokens since they're high-entropy random values.
    ```csharp
    private static string HashToken(string token) =>
        Convert.ToBase64String(SHA256.HashData(Encoding.UTF8.GetBytes(token)));
    ```

    ```csharp
    [Route("api/v1/[controller]")]
    public class AuthController : ControllerBase
    ```

    **Endpoints:**

    1. `POST /api/v1/auth/register` [EnableRateLimiting("public")]
       - Accepts RegisterRequest body
       - Calls _authService.RegisterAsync(request, GetIpAddress())
       - Returns 201 with `{ message: "Conta criada. Verifique seu e-mail para confirmar." }` on success
       - Returns 409 if email already exists
       - Returns 400 with validation errors (FluentValidation auto-validates via pipeline)

    2. `POST /api/v1/auth/login` [EnableRateLimiting("public")]
       - Accepts LoginRequest body
       - Calls _authService.LoginAsync(request, GetIpAddress())
       - On success: set refresh token cookie and return AuthResponse (accessToken + user info)
       - Cookie settings: httpOnly=true, secure=true, sameSite=SameSiteMode.None, expires=7 days, path="/api/v1/auth"
       - After setting cookie, manually append Partitioned attribute for .NET 8:
         ```csharp
         var setCookie = Response.Headers["Set-Cookie"].ToString();
         if (setCookie.Contains("SameSite=None") && !setCookie.Contains("Partitioned"))
         {
             Response.Headers["Set-Cookie"] = setCookie.Replace("SameSite=None", "SameSite=None; Partitioned");
         }
         ```
       - Returns 401 with error message on failure (generic "E-mail ou senha incorretos." — don't reveal which)
       - Returns 429 if account locked (with lockout message from service)

    3. `POST /api/v1/auth/refresh` [EnableRateLimiting("authenticated")]
       - Reads refresh token from Request.Cookies["refreshToken"]
       - Calls _authService.RefreshAsync(token, GetIpAddress())
       - On success: set new refresh token cookie (same settings), return new AuthResponse
       - On failure: clear cookie, return 401

    4. `POST /api/v1/auth/logout` [Authorize] [EnableRateLimiting("authenticated")]
       - Reads refresh token from cookie
       - Calls _authService.LogoutAsync(token, GetIpAddress())
       - Clears the refresh token cookie
       - Returns 200 `{ message: "Sessao encerrada." }`

    5. `POST /api/v1/auth/logout-all` [Authorize] [EnableRateLimiting("authenticated")]
       - Gets userId from JWT claims
       - Calls _authService.LogoutAllAsync(userId)
       - Clears cookie
       - Returns 200 `{ message: "Todas as sessoes encerradas." }`

    6. `POST /api/v1/auth/forgot-password` [EnableRateLimiting("public")]
       - Accepts ForgotPasswordRequest
       - Calls _authService.ForgotPasswordAsync(request.Email)
       - Always returns 200 `{ message: "Se o e-mail existir, enviaremos um link de recuperacao." }` (don't reveal if email exists)

    7. `POST /api/v1/auth/reset-password` [EnableRateLimiting("public")]
       - Accepts ResetPasswordRequest
       - Calls _authService.ResetPasswordAsync(request.Token, request.Email, request.NewPassword)
       - Returns 200 `{ message: "Senha alterada com sucesso." }` or 400 `{ error: "Token invalido ou expirado." }`

    **Helper:**
    ```csharp
    private string GetIpAddress() =>
        HttpContext.Connection.RemoteIpAddress?.MapToIPv4().ToString() ?? "unknown";
    ```

    **Private method for setting refresh token cookie:**
    ```csharp
    private void SetRefreshTokenCookie(string token)
    {
        Response.Cookies.Append("refreshToken", token, new CookieOptions
        {
            HttpOnly = true,
            Secure = true,
            SameSite = SameSiteMode.None,
            Expires = DateTime.UtcNow.AddDays(7),
            Path = "/api/v1/auth"
        });
        // .NET 8 Partitioned attribute workaround
        var setCookie = Response.Headers["Set-Cookie"].ToString();
        if (setCookie.Contains("SameSite=None") && !setCookie.Contains("Partitioned"))
        {
            Response.Headers["Set-Cookie"] = setCookie.Replace("SameSite=None", "SameSite=None; Partitioned");
        }
    }
    ```
